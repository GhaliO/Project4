Downloading Your Programming Platform

Visual Studio Code is a source code editor and a platform that will allow you to code in programming languages like HTML and CSS.

Download Visual Studio Code:

  1. Open your browser and go to the official Visual Studio Code website.
  2. Choose your version for your operating system. Your operating system is either Windows, macOS, or Linux.
  3. Simply click on the download button for your operating system.
  4. Follow the instructions for a step-by-step guide to complete the installation process.
  5. Once the installation has completed, open Visual Studio Code!

Using Visual Studio Code

Creating and Managing Projects:

  1. Open VS Code and click on the "Create a new project" button to open your project folder.
  2. Select the framework type you will be using and the programming language.
  3. Then click “Next” and enter your project name location directory.
  4. Finally, select your target framework and click on “Create”. (The framework is just the prebuilt in code that helps you structure your code instead of starting from scratch)

This will direct you to the source code page where you will write your code. Make sure to head over to the right panel of your project, right-click, and add HTML source code and then CSS source code. This will create two code sources that you will use for your website.

Caution: Different programming languages may have specific naming conventions for variables, functions, classes, and files. Following these conventions ensures that your code is functional and easily understood.

Warning Notice! Before you start programming, it is important to be aware of the potential data loss that you may encounter. Programmers have to be extra careful about taking the appropriate measures to protect their work. Follow these precautions:

  1. Backup Your Files:
    Regularly back up your project files to prevent data loss in case of unexpected events.
  2. Auto-save Settings:
    Configure your code editor to auto-save files at regular intervals to minimize the risk of losing unsaved changes.
  3. Offline Editing:
    If possible, avoid editing critical files while offline to prevent conflicts and data loss when syncing.
